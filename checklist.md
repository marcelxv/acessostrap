## Checklist de Acessibilidade

### Diretriz 1.1 : Fornecer alternativas de texto para todos os conteúdos não-texto
- [ ] 1.1.1 Para todo o conteúdo não-texto,um dos seguintes atendera: [Como atender 1.1.1]	
* Se o conteúdo não-texto apresentar informações ou responder à entrada do usuário, as alternativas de texto servem à mesma finalidade e apresentam as mesmas informações que o conteúdo não-texto. Se as alternativas de texto não puderem servir à mesma finalidade, as alternativas de texto pelo menos identificam a finalidade do conteúdo não-texto.
* Se o conteúdo não-texto for multimídia; conteúdo somente para áudio ao vivo ou somente para vídeo ao vivo; um teste ou exercício que deve usar um sentido específico; ou destinado principalmente a criar uma experiência sensorial específica; em seguida, as alternativas de texto pelo menos identificam o conteúdo não-texto com um rótulode texto descritivo . (Para multimídia, consulte também a diretriz 1.2 Fornecer alternativas sincronizadas para multimídia .)
* Se o objetivo do conteúdo não-texto é confirmar que o conteúdo está sendo operado por uma pessoa em vez de um computador, diferentes formas são fornecidas para acomodar múltiplas deficiências.
* Se o conteúdo não texto é de decoração pura,ou usado apenas para formatação visual, ou se não for apresentado aos usuários, ele é implementado de tal forma que pode ser ignorado pela tecnologia assistiva.

- [x] 1.2.1 Legendas são fornecidas para multimídiapré-gravada . [Como atender 1.2.1]
- [ ] 1.2.2 As audiodescrições do vídeo, ou uma alternativa de texto multimídia completa, incluindo qualquer interação,são fornecidas para multimídia pré-gravada . [Como atender 1.2.2]
- [ ] 1.2.3 As audiodescrições do vídeo são fornecidas para multimídiapré-gravada . [Como atender 1.2.3]
- [ ] 1.2.4 Legendas são fornecidas para multimídia ao vivo . [Como atender 1.2.4]

- [ ] 1.2.6 As descrições de áudio estendidas do vídeo são fornecidas para multimídiapré-gravada . [Como atender 1.2.6]
- [ ] 1.2.7 Para a multimídia pré-gravada, é fornecida uma alternativa de texto multimídia completa, incluindo qualquer interação. [Como atender 1.2.7]

### Diretriz 1.3 : Certifique-se de que as informações e a estrutura possam ser separadas da apresentação
- [ ] 1.3.1 As informações e relacionamentos transmitidos por meio da apresentação podem ser programadase a notificação de alterações a elas está disponível para os agentes do usuário,incluindo tecnologias assistivas. [Como atender 1.3.1]
- [ ] 1.3.2 Qualquer informação transmitida por cor também é visualmente evidente sem cor. [Como atender 1.3.2]
- [ ] 1.3.3 Quando a sequência do conteúdo afeta seu significado, essa sequência pode ser programadaticamente determinada. [Como atender 1.3.3]
- [ ] 1.3.4 As informações que são transmitidas por variações na apresentação do texto também são veiculadas no texto, ou as variações na apresentação do texto podem ser determinadas programáticamente. [Como atender 1.3.4]
- [ ] 1.3.5 As informações necessárias para entender e operar o conteúdo não dependem da forma, tamanho, localização visual ou orientação dos componentes. [Como atender 1.3.5]

### Diretriz 1.4 : Facilitar a distinção de informações em primeiro plano de seu fundo
- [ ] 1.4.1 Texto ou diagramas, e seu plano de fundo, têm uma razão de contraste de luminosidade de pelo menos 5:1. [Como atender 1.4.1]
- [ ] 1.4.2 Um mecanismo está disponível para desligar o áudio de fundo que é reproduzido automaticamente, sem exigir que o usuário desligue todo o áudio. [Como atender 1.4.2]	
- [ ] 1.4.3 Texto ou diagramas, e seu plano de fundo, têm uma razão de contraste de luminosidade de pelo menos 10:1. [Como atender 1.4.3]
- [ ] 1.4.4 O conteúdo de áudio não contém sons de fundo, sons de fundo podem ser desligados ou sons de fundo são pelo menos 20 decibéis inferiores ao conteúdo de áudio em primeiro plano, com exceção de efeitos sonoros ocasionais. [Como atender 1.4.4]
Nota: Uma diferença de 20 decibéis no nível de som é aproximadamente quatro vezes (4x) mais silenciosa ou mais alta. O som de fundo que atender a este requisito será aproximadamente quatro vezes (4x) mais silencioso do que o conteúdo de áudio em primeiro plano.

### Diretriz 2.1 : Torne toda a funcionalidade operável através de uma interface de teclado
- [ ] 2.1.1 Toda a funcionalidade do conteúdo é operável de forma não dependente do tempo através de uma interface de teclado,exceto quando a tarefa requer entrada analógica e dependente do tempo. [Como atender 2.1.1]
Nota: Isso não exclui e não deve desencorajar o suporte de outros métodos de entrada (como um mouse) além da operação do teclado.
- [ ] 2.1.2 Toda a funcionalidade do conteúdo é operável de forma não dependente do tempo através de uma interface de teclado. [Como atender 2.1.2]

### Diretriz 2.2 : Permitir que os usuários controlem os limites de tempo em sua leitura ou interação
- [ ] 2.2.1 Para cada intervalo que é uma função do conteúdo, pelo menos um dos seguintes é verdadeiro: [Como atender 2.2.1]
o usuário pode desativar o tempo de saída; ou
o usuário pode ajustar o tempo de saída em uma ampla faixa que é pelo menos dez vezes maior do que o comprimento da configuração padrão; ou
o usuário é avisado antes do tempo expirar e dá pelo menos 20 segundos para estender o tempo limite com uma ação simples (por exemplo, "aperte qualquer tecla"), e o usuário pode estender o tempo limite pelo menos dez vezes; ou
o tempo-out é uma parte importante de um evento em tempo real (por exemplo, um leilão), e nenhuma alternativa ao tempo-out é possível; ou
o tempo limite faz parte de uma atividade onde o tempo é essencial (por exemplo, jogos competitivos ou testes baseados no tempo) e os prazos não podem ser estendidos ainda mais sem invalidar a atividade.
- [ ] 2.2.2 O conteúdo não pisca por mais de três segundos ou um método está disponível para parar todo o conteúdo piscando na unidade web ou componente autorado. [Como atender 2.2.2]
Nota: Para requisitos relacionados a conteúdo cintilante ou piscando, consulte a Diretriz 2.3 Permita que os usuários evitem conteúdo que possa causar convulsões devido à fotosensibilidade .
- [ ] 2.2.3 O conteúdo pode ser pausado pelo usuário, a menos que o tempo ou movimento faça parte de uma atividade onde o tempo ou o movimento sejam essenciais. [Como atender 2.2.3]
- [ ]	2.2.4 Exceto para eventos em tempo real,o tempo não é uma parte essencial do evento ou atividade apresentada pelo conteúdo. [Como atender 2.2.4]
- [ ]	2.2.5 Interrupções, como conteúdo atualizado, podem ser adiadas ou suprimidas pelo usuário, exceto interrupções envolvendo uma emergência. [Como atender 2.2.5]
- [ ]	2.2.6 Quando uma sessão autenticada expira, o usuário pode continuar a atividade sem perda de dados após a re autenticação. [Como atender 2.2.6]


### Diretriz 2.3 : Permitir que os usuários evitem conteúdo que possa causar convulsões devido à fotosensibilidade
- [ ]	2.3.1 O conteúdo não viola o limiar geral do flash ou o limiar de flash vermelho. [Como atender 2.3.1]
- [ ]	2.3.2 As unidades web não contêm componentes que piscam mais de três vezes em qualquer período de 1 segundo. [Como atender 2.3.2]
### Diretriz 2.4 : Fornecer mecanismos para ajudar os usuários a encontrar conteúdo, orientar-se dentro dele e navegar através dele
- [ ]	2.4.1 Um mecanismo está disponível para contornar blocos de conteúdo que são repetidos em várias unidades da Web. [Como atender 2.4.1]		
- [ ]	2.4.2 Mais de uma maneira está disponível para localizar conteúdo dentro de um conjunto de unidades da Web onde o conteúdo não é o resultado ou um passo dentro de um processo ou tarefa. [Como atender 2.4.2]
- [ ]	2.4.3 Unidades web têm títulos. [Como atender 2.4.3]
- [ ]	2.4.4 Cada link está programadoticamente associado ao texto a partir do qual seu propósito pode ser determinado. [Como atender 2.4.4]
- [ ]	2.4.5 Títulos, títulos e rótulos são descritivos. [Como atender 2.4.5]
- [ ]	2.4.6 Quando uma unidade Web ou componente autorado é navegado sequencialmente, os componentes recebem foco em uma ordem que segue relacionamentos e sequências no conteúdo. [Como atender 2.4.6]
- [ ]	2.4.7 Informações sobre a localização do usuário dentro de um conjunto de unidades da Web estão disponíveis. [Como atender 2.4.7]
- [ ]	2.4.8 O propósito de cada link pode ser programadoticamente determinado a partir do link. [Como atender 2.4.8]

 
### Diretriz 2.5 : Ajude os usuários a evitar erros e facilite a correção de erros que ocorrem	
- [ ]	2.5.1 Se um erro de entrada for detectado, o erro será identificado e descrito ao usuário no texto. [Como atender 2.5.1]	
- [ ]	2.5.2 Se um erro de entrada for detectado e as sugestões de correção forem conhecidas e forem fornecidas sem comprometer a segurança ou a finalidade do conteúdo, as sugestões são fornecidas ao usuário. [Como atender 2.5.2]
- [ ]	2.5.3 Para que ocorram transações legais ou financeiras, que modifiquem ou excluam dados em sistemas de armazenamento de dados ou que enviem respostas de teste, pelo menos uma das seguintes é verdadeira: [Como atender 2.5.3]
Ações são reversíveis.
As ações são verificadas para erros de entrada antes de passar para a próxima etapa do processo.
O usuário pode revisar e confirmar ou corrigir informações antes de submetê-la.
- [ ]	2.5.4 A ajuda sensível ao contexto está disponível para entrada de texto. [Como atender 2.5.4]

 
### Diretriz 3.1 : Torne o conteúdo do texto legível e compreensível.
- [ ]	3.1.1 A linguagem natural primária ou as línguas da unidade Web podem ser programadasticamente determinadas. [Como atender 3.1.1]	
- [ ]	3.1.2 A linguagem natural de cada passagem ou frase na unidade web pode ser programadaticamente determinada. [Como atender 3.1.2]
Nota: Esse requisito não se aplica a palavras ou frases individuais que se tornaram parte da linguagem primária do conteúdo.		
- [ ]	3.1.3 Um mecanismo está disponível para identificar definições específicas de palavras ou frases usadas de forma incomum ou restrita, incluindo expressões e jargões. [Como atender 3.1.3]	
- [ ]	3.1.4 Um mecanismo para encontrar a forma expandida de abreviaturas está disponível. [Como atender 3.1.4]
- [ ]	3.1.5 Quando o texto exige capacidade de leitura mais avançada do que o nível médio mais baixo,está disponível conteúdo suplementar que não exige capacidade de leitura mais avançada do que o nível de ensino médio mais baixo. [Como atender 3.1.5]
- [ ]	3.1.6 Um mecanismo está disponível para identificar pronúncia específica de palavras onde o significado não pode ser determinado sem pronúncia. [Como atender 3.1.6]

 
### Diretriz 3.2 : Torne previsível a colocação e funcionalidade do conteúdo.		
- [ ]	3.2.1 Quando qualquer componente recebe foco, não causa uma mudança de contexto. [Como atender 3.2.1]
- [ ]	3.2.2 Alterar a configuração de qualquer controle de formulário ou campo não causa automaticamente uma mudança de contexto (além de passar para o próximo campo em ordem de guia), a menos que a unidade de autoria contenha instruções antes do controle que descreva o comportamento. [Como atender 3.2.2]
- [ ]	3.2.3 Mecanismos de navegação que são repetidos em várias unidades da Web dentro de um conjunto de unidades Web ou outros recursos primários ocorrem na mesma ordem relativa cada vez que são repetidos, a menos que uma alteração seja iniciada pelo usuário. [Como atender 3.2.3]
- [ ]	3.2.4 Os componentes que possuem a mesma funcionalidade dentro de um conjunto de unidades Web são identificados de forma consistente. [Como atender 3.2.4]
- [ ]	3.2.5 As alterações de contexto são iniciadas apenas por solicitação do usuário. [Como atender 3.2.5]

 
### Diretriz 4.1 : Suporte a compatibilidade com os atuais e futuros agentes do usuário (incluindo tecnologias assistivas)
- [ ]	4.1.1 Unidades web ou componentes autorais podem ser analisados de forma inequívoca, e as relações na estrutura de dados resultante também são inequívocas. [Como atender 4.1.1]
- [ ]	4.1.2 Para todos os componentes da interface do usuário, o nome e a função podem ser programados,valores que podem ser definidos pelo usuário podem ser definidos programáticamente,e a notificação de alterações nesses itens está disponível para os agentes do usuário,incluindo tecnologias assistivas. [Como atender 4.1.2]

O conteúdo está em conformidade com o critério de sucesso 2.3.1 (flash geral e vermelho).
- [ ]	4.2.3 Pelo menos uma versão do conteúdo atende a todos os critérios de sucesso nível 2, mas as versões alternativas que não atendem a todos os critérios de sucesso nível 2 podem estar disponíveis a partir do mesmo URI. [Como atender 4.2.3]
- [ ]	4.2.4 O conteúdo implementado utilizando tecnologias fora da linha de base escolhida satisfaz todos os requisitos de Nível 1 e Nível 2 suportados pelas tecnologias. [Como atender 4.2.4]
